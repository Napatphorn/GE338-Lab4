### แบบจำลองพื้นที่เสี่ยงภัยดินโคลนถล่ม
### กรณีศึกษา: จังหวัดเชียงใหม่

## ภารกิจที่ 1: กำหนดกรอบแนวคิด

**คำถามวิจัย:** "พื้นที่ใดในจังหวัดเชียงใหม่ที่มีความเปราะบางต่อการเกิดดินโคลนถล่มสูงที่สุด โดยพิจารณาจากปัจจัยทางกายภาพ ทางอุทกวิทยา และอิทธิพลของสิ่งปกคลุมดิน?"
แบบจำลองนี้คัดเลือกปัจจัยที่มีอิทธิพลต่อเสถียรภาพของความลาดชัน

1.  **ความลาดชัน (Slope):** เป็นปัจจัยที่สำคัญที่สุด ดินจะพังทลายลงมาได้ต้องมีแรงขับเคลื่อน จากน้ำหนักมวลดินขนานกับแนวชัน หากพื้นที่ราบเรียบ แรงนี้จะเป็นศูนย์
2.  **ปริมาณน้ำฝน (Rainfall):** เป็นตัวชนวน ที่ทำให้น้ำเข้าไปแทนที่อากาศในช่องว่างระหว่างเม็ดดิน จนลดแรงต้านทานการตัดของดิน
3.  **ดัชนีพืชพรรณ (NDVI):** รากพืชช่วยยึดเกาะหน้าดิน และเรือนยอดช่วยชะลอพลังงานจลน์ของเม็ดฝนที่ตกกระทบผิวดิน
4.  **ระยะห่างจากแหล่งน้ำ (Distance to Water):** เป็นปัจจัยด้านการกัดเซาะ กระแสน้ำในลำห้วยจะกัดเซาะบริเวณฐานของภูเขา ทำให้ดินส่วนบนสูญเสียแรงพยุงและถล่มลงมา

## ภารกิจที่ 2: การสร้างโมเดลและที่มาของน้ำหนัก

แบบจำลองนี้อ้างอิงค่าน้ำหนักที่ผ่านการคำนวณด้วยวิธี **Analytic Hierarchy Process (AHP)** จากงานวิจัยของ **แพรวา วิจิตรธนสาร และ อนุเผ่า อบแพทย์ (2565)** เรื่อง *"การพัฒนาแผนที่ความเสี่ยงภัยพิบัติทางธรรมชาติในประเทศไทย"* ซึ่งเป็นการศึกษาในบริบททางธรณีวิทยาของไทยโดยเฉพาะ
### ค่าน้ำหนักและความสำคัญ (Weighting Scale)
| ปัจจัย (Factor) | น้ำหนัก (Weight) | เหตุผลตามงานวิจัยอ้างอิง |
| :--- | :---: | :--- |
| **ความลาดชัน (Slope)** | **0.40 (40%)** | เป็นปัจจัยหลักที่กำหนดโอกาสการเกิดตามหลักวิศวกรรมโยธา |
| **ปริมาณน้ำฝน (Rainfall)** | **0.30 (30%)** | เป็นปัจจัยกระตุ้นอันดับหนึ่งสำหรับพื้นที่มรสุมอย่างภาคเหนือ |
| **ดัชนีพืชพรรณ (NDVI)** | **0.15 (15%)** | บทบาทของการปกคลุมดินในการลดทอนความรุนแรง |
| **ระยะห่างจากแหล่งน้ำ** | **0.15 (15%)** | ปัจจัยเสริมจากการกัดเซาะทางอุทกวิทยา |

**กระบวนการทางคณิตศาสตร์ใน Google Earth Engine:**
1.  **Normalization:** ปรับสเกลข้อมูลทุกชั้น (0 - 1) โดยใช้ฟังก์ชัน `(x - min) / (max - min)`
2.  **Inversion:** สำหรับ NDVI และ Distance to Water ได้ทำการกลับค่าด้วยสมการ `1 - x` เพื่อให้พื้นที่ที่มีพืชพรรณน้อยและพื้นที่ที่ใกล้แหล่งน้ำ ได้คะแนนความเสี่ยงที่สูงขึ้น
3.  **Weighting:** รวมผลลัพธ์ด้วยวิธี Weighted Linear Combination (WLC) เพื่อสร้างแผนที่ความเสี่ยงฉบับสมบูรณ์


## ภารกิจที่ 3: การวิเคราะห์ความอ่อนไหว (Sensitivity Analysis)

จากการทดสอบปรับเปลี่ยนค่าน้ำหนัก (Sensitivity Test) เพื่อตรวจสอบความเชื่อมั่นของโมเดล พบผลลัพธ์ดังนี้

* **ความอ่อนไหวต่อ Slope:** เมื่อลดน้ำหนัก Slope ลง พื้นที่เสี่ยงสีแดงจะหายไปจากบริเวณหุบเขาเกือบทั้งหมด แม้จะมีฝนตกหนักก็ตาม ยืนยันว่า Slope คือ ปัจจัยที่สำคัญที่สุดของโมเดลนี้
* **อิทธิพลของ NDVI:** ในบริเวณรอยต่อเขตป่าและพื้นที่เกษตรกรรม ผลลัพธ์มีความอ่อนไหวต่อค่า NDVI สูงมาก หากพืชพรรณลดลงเพียงเล็กน้อย ระดับความเสี่ยงจะดีดตัวขึ้นทันที
* **ความคงตัวของผลลัพธ์ (Robustness):** พื้นที่ภูเขาสูงใน อ.เชียงดาว และ อ.อมก๋อย แสดงผลความเสี่ยงสูงอย่างต่อเนื่องในทุกชุดค่าน้ำหนัก เนื่องจากมีค่าความชันและปริมาณฝนที่สูงเกินค่าเกณฑ์มาตรฐาน

## ภารกิจที่ 4: การตรวจสอบความถูกต้อง (Validation)

**ข้อจำกัดและการแก้ไข:** เนื่องจากฐานข้อมูล **NASA Global Landslide Catalog** บนระบบ API เกิดปัญหาการเข้าถึงผู้วิจัยจึงใช้วิธี **"Visual Proxy Validation"** โดยมีกระบวนการดังนี้:

1.  **การซ้อนทับข้อมูล (Overlay):** นำพื้นที่ที่โมเดลระบุว่า เสี่ยงสูงมาก (Score > 0.45) ซ้อนทับบนภาพถ่ายดาวเทียมความละเอียดสูง
2.  **การตีความผล:** พบว่าพื้นที่สีแดงกระจุกตัวอยู่ในบริเวณ **"ร่องเขา (Gully)"** และ **"ไหล่เขาชัน"** อย่างแม่นยำ ซึ่งตามหลักธรณีสัณฐานวิทยาคือจุดเสี่ยงที่สุดในการเกิด Debris Flow
3.  **การตรวจสอบร่องรอยจริง:** ในเขตพื้นที่ อ.แม่ริม พบว่าจุดสีแดงตรงกับบริเวณที่มีหน้าดินเปิดโล่งและไม่มีต้นไม้ใหญ่ปกคลุม ซึ่งสอดคล้องกับพฤติกรรมการเกิดดินถล่มจริงในอดีต


## ตอบคำถามสรุปผล

**1. ถ้ามีข้อมูล Ground Truth จริง คิดว่า Accuracy จะอยู่ในระดับใด?**
* **ตอบ:** คาดว่าความแม่นยำ (Overall Accuracy) จะอยู่ที่ประมาณ **65-70%** เนื่องจากปัจจัยหลักสอดคล้องกับหลักวิศวกรรมในงานวิจัยอ้างอิง แต่ความคลาดเคลื่อนจะมาจากความละเอียดของข้อมูลฝน ที่มีขนาดพิกเซลหยาบกว่าข้อมูลภูมิประเทศ

**2. ปัจจัยที่ไม่ได้ใส่ในโมเดลแต่น่าจะสำคัญมีอะไรบ้าง?**
* **ตอบ:** **"ธรณีวิทยา (Geology)"** และ **"ชนิดของดิน (Soil Texture)"** เช่น หินดินดาน หรือดินร่วนปนทรายจะพังทลายง่ายกว่าหินแกรนิต สาเหตุที่ไม่ใส่เนื่องจากข้อมูล Digital Soil Map ของไทยในระบบ GEE ยังมีความละเอียดไม่เพียงพอต่อการวิเคราะห์ระดับพื้นที่ในปัจจุบัน

**3. โมเดลนี้ใช้ได้ดีในระดับ Scale ใด?**
* **ตอบ:** ระดับ **"อำเภอ หรือ จังหวัด"** เพราะข้อมูลปริมาณน้ำฝนและ NDVI มีความละเอียดประมาณ 1 กม. ไม่แนะนำให้ใช้ในระดับตำบลหรือรายบ้าน เพราะอาจเกิดความผิดพลาดในการระบุตำแหน่งที่แน่นอนได้

**4. ขั้นตอนการอัปเดตโมเดลรายปี?**
* **ตอบ:** * **ส่วนที่ต้องอัปเดต:** ปริมาณน้ำฝน (Rainfall) และดัชนีพืชพรรณ (NDVI) เนื่องจากเปลี่ยนแปลงตามปีที่ศึกษา
    * **ส่วนที่คงที่ (Static):** ความลาดชัน (Slope) และระยะห่างจากแหล่งน้ำ (Distance to Water) ซึ่งเป็นลักษณะทางกายภาพที่ไม่เปลี่ยนแปลงในระยะเวลาสั้นๆ
